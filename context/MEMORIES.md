## Key Decisions Made

### Architecture Decisions

1. **Three-tier model loading** (2026-04-23)
   - Remote URL (`MODELS_REGISTRY_URL`) → Local file (`public/models.json`) → `BUNDLED_MODELS`
   - Reason: Without `MODELS_REGISTRY_URL` env var, models.json was dead. Local file read ensures GitHub Actions auto-updates actually take effect.

2. **META prefix matching** (2026-04-23)
   - `lookupMeta(id)` tries exact match first, then longest-prefix match
   - Reason: One META entry for "gpt-5" covers gpt-5, gpt-5-turbo, gpt-5-0513, etc.

3. **Category-based model separation** (2026-04-23)
   - Generator models: `category === "text"` only (text/reasoning models)
   - Target models: ALL categories (text, image, video, tts, stt, embedding, ocr)
   - Reason: Generator writes prompts (needs text capability). Target can be anything.

4. **scoreModel with 4 modes** (2026-04-23)
   - `token` (cheapest), `fast` (speed), `accurate` (quality), `aligned` (balanced)
   - Generator auto-selection: uses "token" for text targets, "accurate" for image/video targets
   - Reason: Image prompts need quality, text prompts can use cheaper models.

5. **AihubMix as primary relay** (2026-04-23)
   - Custom relay flow: user enters base URL + API key → system probes `/v1/models` → discovers available models → auto-selects best generator
   - Probe result cached in localStorage with 1-hour TTL

### What NOT To Do

1. **Never hardcode model lists** — use `public/models.json` as source of truth
2. **Never require `MODELS_REGISTRY_URL`** — local file reading is sufficient
3. **Don't filter out zero-cost models** — they're valid models, just missing pricing data
4. **Don't use `META[m.id]` directly** — always use `lookupMeta(m.id)` for prefix matching

## Technical Gotchas

1. **AihubMix `m.created` is fake**: Always returns 1626739200 (2021-07-20). Don't use it for release dates. Use META's `d` field instead.
2. **Rebase during auto-update conflicts**: GitHub Actions commits to `models.json` every 2 hours. If you're working on code that changes `models.json`, use `git pull --no-rebase -X ours` to merge, then re-run `node scripts/patch-models.cjs`.
3. **SSL/TLS errors on push**: Sometimes the first `git push` fails with `schannel: failed to receive handshake`. Just retry.
4. **`public/sw.js` and related files**: Service worker files generated by next-pwa. Appear as untracked/modified. Not critical — can be ignored or committed separately.
