# Quick Start for AI Assistants

> Read this to get up to speed in 30 seconds.
> **IMPORTANT**: Before doing any work, also read `context/PROGRESS.md` for current task status.

## Project

AI prompt optimizer web app. User inputs rough idea → system generates optimized prompt for a target LLM.

**Stack**: Next.js 14 + TypeScript + Tailwind + Framer Motion. Deployed on Vercel.
**Site**: https://www.myprompt.asia
**Repo**: https://github.com/00qq6868-del/ai-prompt-generator.git

## Context Files (READ THESE FIRST)

| File | What it tells you |
|------|------------------|
| `context/PROGRESS.md` | What's done, what's pending, what to do next |
| `context/MEMORIES.md` | All decisions, user preferences, technical gotchas |
| `context/SESSION_LOG.md` | What each AI session did — read the latest entry |
| `context/SYSTEM_STATE.json` | Current model counts (auto-generated by CI) |

## Critical Code Files

| File | Purpose |
|------|---------|
| `src/lib/model-cache.ts` | Server-side cache: remote URL → local `public/models.json` → BUNDLED_MODELS |
| `.github/scripts/fetch-models.mjs` | Auto-fetches 250+ models from APIs, writes `public/models.json` |
| `src/lib/models-registry.ts` | `ModelInfo` type, `scoreModel()`, `BUNDLED_MODELS` fallback |
| `src/components/PromptGenerator.tsx` | Main orchestrator — auto-selects generator, calls `/api/generate` |
| `src/components/ModelSelector.tsx` | Model selection UI with category/provider filters |
| `src/lib/providers/index.ts` | 13 LLM provider adapters |
| `src/lib/prompt-optimizer.ts` | System prompt templates for prompt generation |
| `scripts/patch-models.cjs` | Post-process models.json (apply META + classify) |
| `scripts/latest-model-ensures.cjs` | Verified latest-model fallback list for OpenAI/DeepSeek/Gemini/Claude |

## Dev Commands

```bash
npm run dev      # Start dev server (port 3000)
npm run build    # Production build
npm run test:quality # Visual-health + accessibility audit
npx tsc --noEmit # Type check only
node scripts/patch-models.cjs  # Re-apply META data to models.json
```

## Key Patterns

- **META + lookupMeta()**: Model metadata with prefix-matching fallback
- **scoreModel(model, mode)**: 4 scoring modes — `token`, `fast`, `accurate`, `aligned`
- **Probe flow**: KeysSettings saves relay config → calls `/api/probe` → caches model list → auto-selects generator
- **Bilingual errors**: All user-facing messages in `中文 / English` format
- **Category-aware routing**: Generator uses `accurate` mode for image/video targets, `token` for text
- **Generator = text only, Target = all categories** (image, video, tts, etc.)
- **Model auto-update**: GitHub Actions polls provider APIs every 30 minutes, then applies `latest-model-ensures.cjs`

## Before You End Your Session

1. Update `context/PROGRESS.md` with what you completed and what's pending
2. Add a new entry to `context/SESSION_LOG.md`
3. Run `bash scripts/save-context.sh` to commit and push context to GitHub
4. If the script doesn't work, manually: `git add context/ && git commit -m "chore: update context" && git push`
